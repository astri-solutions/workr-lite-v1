Referência de HTML — Workr Lite

Classes, estruturas e atributos disponíveis para uso em matérias do tipo HTML. Aplique apenas em páginas e canais do tipo Show.

Somente tipo Show Prefixo wl- Sem !important
ℹ Todo o HTML colado em uma matéria é renderizado dentro de um container .wl-materia já provido pelo portal. Não inclua <html>, <head> ou <body> — inicie diretamente com as seções do conteúdo.
01 Estrutura base

Todo conteúdo HTML é composto por seções (wl-section) que contêm um container interno (wl-container). Esta hierarquia é obrigatória para que o layout fique consistente com o restante do portal.

.wl-section + .wl-container Bloco estrutural fundamental de toda matéria HTML 
<!-- Seção padrão -->
<section class="wl-section">
  <div class="wl-container">
    <!-- conteúdo aqui -->
  </div>
</section>

<!-- Seção com fundo alternado (cinza claro) -->
<section class="wl-section wl-section--alt">
  <div class="wl-container">
    ...
  </div>
</section>

<!-- Seção com fundo teal (destaque) -->
<section class="wl-section wl-section--teal">
  <div class="wl-container">
    ...
  </div>
</section>
ClasseModificadorDescrição
.wl-sectionbaseBloco de seção com padding vertical padrão
.wl-section--altmodFundo cinza claro (#F4F4F4)
.wl-section--tealmodFundo teal escuro, texto branco
.wl-section--smmodPadding vertical reduzido (metade)
.wl-section--lgmodPadding vertical ampliado
.wl-containerbaseContainer centralizado (max-width 1080px)
.wl-container--narrowmodContainer estreito (max-width 720px) — ideal para texto longo
.wl-container--widemodContainer ampliado (max-width 1280px)
02 Tipografia

Headings usam Plus Jakarta Sans. Corpo de texto usa Inter. Aplique as classes diretamente nos elementos semânticos (h1–h4, p, span).

.wl-heading-1 — .wl-heading-4 Hierarquia de títulos 
<h1 class="wl-heading-1">Título principal</h1>
<h2 class="wl-heading-2">Título de seção</h2>
<h3 class="wl-heading-3">Subtítulo</h3>
<h4 class="wl-heading-4">Subtítulo menor</h4>
.wl-lead · .wl-body · .wl-caption · .wl-label Variações de corpo de texto 
<!-- Lead: texto introdutório maior -->
<p class="wl-lead">Texto de destaque, maior que o corpo padrão.</p>

<!-- Body: parágrafo padrão -->
<p class="wl-body">Conteúdo principal da matéria.</p>

<!-- Caption: legenda ou nota de rodapé -->
<p class="wl-caption">Fonte: ITR 2T25 — IMC S.A.</p>

<!-- Label: rótulo em maiúsculo (eyebrow) -->
<span class="wl-label">Resultados Financeiros</span>
.wl-blockquote Citação em destaque com borda lateral teal 
<blockquote class="wl-blockquote">
  "O crescimento de 12% no EBITDA reflete a execução consistente
  de nossa estratégia de expansão regional."
  <cite>Carlos Souza, CFO</cite>
</blockquote>

<div class="wl-cols wl-cols--2">

<!-- Três colunas -->

<!-- Coluna + imagem (2/3 + 1/3) -->

09 Accordion

Recolhimento progressivo de conteúdo. Ideal para FAQs, perguntas de acionistas e documentos agrupados. A animação é controlada por CSS.

.wl-accordion + .wl-accordion__item Grupo de itens expansíveis 
<div class="wl-accordion">

  <div class="wl-accordion__item">
    <button
      class="wl-accordion__trigger"
      data-wl-accordion="toggle"
      aria-expanded="false"
    >
      Qual a política de dividendos da companhia?
    </button>
    <div class="wl-accordion__body">
      <p class="wl-body">A política prevê distribuição mínima de 25% do lucro líquido ajustado.</p>
    </div>
  </div>

  <div class="wl-accordion__item">
    <button
      class="wl-accordion__trigger"
      data-wl-accordion="toggle"
      aria-expanded="false"
    >
      Como participar das teleconferências de resultados?
    </button>
    <div class="wl-accordion__body">
      <p class="wl-body">As instruções de acesso são divulgadas 48h antes via e-mail para acionistas cadastrados.</p>
    </div>
  </div>

</div>
O atributo data-wl-accordion="toggle" é detectado automaticamente pelo script do portal. Não é necessário adicionar JavaScript próprio — o portal inicializa todos os accordions na página.

12 Embeds de dados (CVM / B3)

O portal inclui um módulo de dados financeiros que consome a API CVM e B3. Use os atributos data-wl-* para configurar widgets embutidos — nenhum JavaScript extra é necessário.

⚠ Os embeds de dados requerem que o portal tenha um código CVM configurado em Informações do Portal. Sem esse código, o widget exibirá uma mensagem de configuração pendente.
data-wl-embed="cvm-doc" Lista de documentos CVM (ITR, DFP, Fato Relevante, etc.) 
<!-- Lista os últimos N documentos de um tipo -->
<div
  data-wl-embed="cvm-doc"
  data-wl-tipo="ITR"
  data-wl-limite="5"
  data-wl-layout="lista"
></div>

<!-- Tipos disponíveis: ITR | DFP | FR | CA | AGO | AGE -->
<!-- Layouts: lista | cards | tabela -->
data-wl-embed="cotacao" Cotação em tempo real do papel (B3) 
<!-- Widget de cotação (usa o ticker configurado no portal) -->
<div
  data-wl-embed="cotacao"
  data-wl-ticker="MEAL3"
  data-wl-layout="card"
></div>

<!-- Layout inline: exibe apenas o preço e variação -->
<span
  data-wl-embed="cotacao"
  data-wl-ticker="MEAL3"
  data-wl-layout="inline"
></span>
data-wl-embed="release" Preview do release de resultados mais recente 
<div
  data-wl-embed="release"
  data-wl-periodo="2T25"
  data-wl-layout="destaque"
></div>
data-wl-embed="grafico" Gráfico de histórico de cotação (canvas) 
<div
  data-wl-embed="grafico"
  data-wl-ticker="MEAL3"
  data-wl-periodo="12m"
  data-wl-altura="320"
></div>

<!-- Períodos: 1m | 3m | 6m | 12m | 24m | 60m -->
13 Scripts e data-attrs do portal

O portal executa automaticamente scripts ao detectar determinados atributos no HTML da matéria. Não inclua <script> tags próprias — use os atributos abaixo.

⚠ Tags <script> inseridas manualmente são removidas por segurança antes da publicação. Use exclusivamente os atributos data-wl-* listados aqui.
AtributoValorComportamento
data-wl-accordion"toggle"Inicializa expand/collapse com animação CSS
data-wl-embedver seção 12Monta widget de dados no elemento
data-wl-lazy"true"Carrega imagem ou embed somente ao entrar no viewport
data-wl-lightbox"true"Abre imagem em overlay ao clicar
data-wl-copytextoCopia o valor do atributo ao clipboard no click
data-wl-print"true"Botão que dispara window.print() com layout de impressão
data-wl-downloadURLBaixa o arquivo da URL indicada (rastreado no analytics)
data-wl-videoURL (YouTube/Vimeo)Substitui elemento por player embed responsivo
data-wl-video Player de vídeo embutido responsivo 
<!-- Substitui pelo iframe do YouTube/Vimeo com aspect-ratio 16/9 -->
<div
  data-wl-video="https://www.youtube.com/watch?v=XXXXXXXXXXX"
  data-wl-lazy="true"
></div>

15 Tokens CSS disponíveis

As propriedades custom do portal ficam disponíveis dentro do HTML da matéria via var(). Use-as em style inline ou em blocos <style> escopados.

ℹ Blocos <style> são permitidos desde que os seletores sejam escopados com .wl-materia como ancestral, para evitar vazamento de estilos para o restante do portal.
var(--wl-*) Tokens de cor, tipografia e espaçamento do portal 
/* Cores da marca */
--wl-color-teal:          #0B5B68;
--wl-color-teal-dark:     #084953;
--wl-color-teal-50:       #E8F4F6;
--wl-color-green:         #00D865;
--wl-color-dark:          #141414;
--wl-color-gray-1:        #6F6F6F;
--wl-color-gray-2:        #949494;
--wl-color-gray-3:        #B8B8B8;
--wl-color-bg:            #F4F4F4;

/* Tipografia */
--wl-font-heading:        'Plus Jakarta Sans', sans-serif;
--wl-font-body:           'Inter', sans-serif;

/* Raios de borda */
--wl-radius-sm:           6px;
--wl-radius-md:           10px;
--wl-radius-lg:           16px;

/* Sombras */
--wl-shadow-sm:           0 1px 3px rgba(0,0,0,.06), 0 1px 2px rgba(0,0,0,.04);
--wl-shadow-md:           0 4px 12px rgba(0,0,0,.08);

/* Exemplo de uso em style inline */
<div style="background: var(--wl-color-teal-50); border-radius: var(--wl-radius-md); padding: 20px">
  ...
</div>
